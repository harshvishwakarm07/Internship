# Data Flow Diagram (DFD) - SITS

This document contains DFD Level 0, Level 1, and Level 2 for the Student Internship Tracking System (SITS).

## DFD Level 0 (Context Diagram)

```mermaid
flowchart LR
    S[Student]
    F[Faculty/Mentor]
    A[Admin]
    C[Company optional]

    SYS((SITS System))

    S -->|Register/Login, Internship Data, Reports, Files| SYS
    SYS -->|Status, Feedback, Dashboard Data| S

    F -->|Review Actions, Feedback, Approval/Rejection| SYS
    SYS -->|Assigned Students, Internship/Report Data| F

    A -->|User Management, Mentor Assignment, Admin Actions| SYS
    SYS -->|Analytics, User Lists, Audit View| A

    C -->|Offer/Certificate Confirmation optional| SYS
    SYS -->|Verification Requests optional| C
```

---

## DFD Level 1 (Major Process Decomposition)

```mermaid
flowchart TB
    %% External Entities
    S[Student]
    F[Faculty]
    A[Admin]

    %% Processes
    P1((1.0 Authentication & RBAC))
    P2((2.0 Internship Management))
    P3((3.0 Report & Feedback Management))
    P4((4.0 Admin & Mentor Management))
    P5((5.0 Notifications & Dashboard Analytics))

    %% Data Stores
    D1[(D1 User)]
    D2[(D2 StudentProfile)]
    D3[(D3 Internship)]
    D4[(D4 Report)]
    D5[(D5 Feedback)]
    D6[(D6 AuditLog)]

    %% Flows: Auth
    S -->|register/login data| P1
    F -->|login data| P1
    A -->|login data| P1
    P1 <--> D1
    P1 -->|auth token + role| S
    P1 -->|auth token + role| F
    P1 -->|auth token + role| A

    %% Flows: Internship
    S -->|internship form + offer letter + certificate| P2
    F -->|approve/reject action| P2
    A -->|status override optional| P2
    P2 <--> D3
    P2 -->|status updates| S
    P2 --> D6

    %% Flows: Reports
    S -->|weekly report + attachment| P3
    F -->|report feedback + review| P3
    P3 <--> D4
    P3 <--> D5
    P3 -->|feedback/review status| S
    P3 --> D6

    %% Flows: Admin
    A -->|user CRUD + mentor assignment| P4
    P4 <--> D1
    P4 <--> D2
    P4 <--> D3
    P4 --> D6

    %% Flows: Analytics/Notifications
    P5 <--> D1
    P5 <--> D3
    P5 <--> D4
    P5 <--> D5
    P5 <--> D6
    P5 -->|dashboard metrics + alerts| A
    P5 -->|review queue + alerts| F
    P5 -->|status alerts| S
```

## DFD Level 2 (Process 2.0 Internship Management Decomposition)

```mermaid
flowchart TB
    S[Student]
    F[Faculty]
    A[Admin]

    P21((2.1 Submit Internship))
    P22((2.2 Validate Internship Data))
    P23((2.3 Store Internship + Documents))
    P24((2.4 Review Internship))
    P25((2.5 Update Status Workflow))
    P26((2.6 Publish Status & Notify))

    D3[(D3 Internship)]
    D6[(D6 AuditLog)]

    S -->|company, role, dates, offer letter| P21
    P21 --> P22
    P22 -->|valid| P23
    P22 -->|invalid error| S

    P23 --> D3
    P23 --> D6

    F -->|approve/reject + reason| P24
    A -->|approve/reject + reason optional| P24
    P24 --> P25

    P25 -->|Pending -> Approved/Rejected| D3
    P25 --> D6
    P25 --> P26

    P26 -->|status update + feedback| S
```

## DFD Level 2 (Process 3.0 Report & Feedback Management Decomposition)

```mermaid
flowchart TB
    S[Student]
    F[Faculty]

    P31((3.1 Submit Weekly Report))
    P32((3.2 Validate Report Data))
    P33((3.3 Store Report + Attachment))
    P34((3.4 Review Report))
    P35((3.5 Save Feedback))
    P36((3.6 Update Report Status & Notify))

    D4[(D4 Report)]
    D5[(D5 Feedback)]
    D6[(D6 AuditLog)]

    S -->|weekNumber, content, attachment| P31
    P31 --> P32
    P32 -->|valid| P33
    P32 -->|validation error| S

    P33 --> D4
    P33 --> D6

    F -->|feedback + rating| P34
    P34 --> P35
    P35 --> D5
    P35 --> D6

    P35 --> P36
    P36 -->|Report status: Submitted/Reviewed| D4
    P36 -->|feedback visible on dashboard| S
```

## Notes for Viva/Documentation

1. Level 0 shows only external entities and one central process.
2. Level 1 breaks SITS into major business processes.
3. Level 2 gives detailed subprocess data flow for Internship and Report modules.
4. Data stores map directly to MongoDB models:
   - User
   - StudentProfile
   - Internship
   - Report
   - Feedback
   - AuditLog
